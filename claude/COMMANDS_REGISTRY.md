# Commands Registry

本文件定义 `~/.claude/commands/` 的推荐入口层级，目标是减少重复入口，而不破坏兼容性。

---

## A 级：默认主入口

这些入口应该被优先使用，构成日常主系统：

- `main`
  - 老板总入口
  - 先走 `hermes-loop`，再决定是否拉 `lobster`
- `arch`
  - 架构、技术边界、系统设计
- `pm`
  - 需求、优先级、PRD、策略判断
- `dev`
  - 编码实现、调试、重构、测试
- `ui`
  - 交互、视觉、信息结构、体验判断
- `ops`
  - 运行稳定性、巡检、告警、服务故障处理
- `lobster`
  - 方向挑战、范围收敛、优先级去水分

推荐默认路由：

`main -> hermes-loop -> 是否调用 lobster -> arch/pm/dev/ui/ops -> 具体 skill`

---

## B 级：保留，但默认不主推

这些入口有独立价值，但不应和主入口并列当作第一选择：

- `content`
  - 适合内容生产、SEO、内容分发
  - 默认由 `main` 或 `pm` 判断后再进入
- `finance`
  - 适合账务、税务、成本和财务分析
- `growth`
  - 适合增长实验、漏斗优化、渠道分析
- `legal`
  - 适合法务、合规、条款风险识别
- `support`
  - 适合客服话术、FAQ、反馈汇总
- `hindsight`
  - 这不是角色入口，而是记忆系统维护入口

规则：

- 这些入口保留
- 但不作为对外“主菜单”
- 优先由 `main` 识别任务后再决定是否调用

---

## C 级：降级为别名或内部维护命令

这些入口不建议继续作为一级角色入口使用：

- `ba`
  - 视为 `pm` 的分析子模式
  - 建议：保留兼容，但默认改用 `pm`
- `sm`
  - 视为 `pm` 的敏捷推进子模式
  - 建议：保留兼容，但默认改用 `pm`
- `merge-skill`
  - 这是维护工具，不是角色命令
  - 建议：保留给系统维护使用，不面向日常任务暴露

---

## 实施策略

采用“软收口”，不直接删命令：

1. 保留现有文件，避免破坏旧习惯
2. 在降级入口文件头部明确推荐替代入口
3. 在主注册表中把 `main / arch / pm / dev / ui / ops / lobster` 定为默认集合
4. 后续如果长期不用，再考虑真的移除
