# Office Hours — 创业诊断 & 头脑风暴

## 两种模式

### 创业模式（Startup Mode）
适用于：创业项目、内部创业

**六个逼问（逐个问，等回答后再问下一个）：**

1. **需求现实** — 最强的证据是什么，证明有人真正想要这个？（不是"感兴趣"，是行为、付费）
2. **现状替代** — 用户现在怎么解决这个问题？替代方案的成本是什么？
3. **精确画像** — 说出最需要这个产品的那个人的名字。什么让 TA 升职/被开？
4. **最窄切入** — 这周就有人愿意付费的最小版本是什么？
5. **观察意外** — 你看过用户在无帮助下使用产品吗？什么让你意外？
6. **未来适配** — 3 年后世界变了，这个产品更关键还是更无关？

**智能路由**：
- 还没产品 → Q1, Q2, Q3
- 有用户 → Q2, Q4, Q5
- 有付费用户 → Q4, Q5, Q6

### 建设者模式（Builder Mode）
适用于：副项目、黑客松、学习、开源

**探索性提问：**
- 最酷的版本长什么样？
- 你会展示给谁？什么让他们说"哇"？
- 最快能用/能分享的路径？
- 最接近的现有产品是什么？你的有什么不同？
- 无限时间的 10x 版本？

## 前提挑战（两种模式都要）
1. 这是对的问题吗？换个角度是否更简单？
2. 不做会怎样？真痛还是假痛？
3. 现有代码已经部分解决了什么？

## 方案生成（必须）
2-3 个不同方案：

```
方案 A: {名称}
  摘要: {1-2 句}
  工作量: S/M/L/XL
  风险: 低/中/高
  优点: {2-3 条}
  缺点: {2-3 条}
```

规则：至少一个最小可行方案 + 一个理想架构方案。

## 输出：设计文档
写入项目根目录 `design-{日期}.md`，包含：
- 问题陈述
- 需求证据 / 酷在哪里
- 约束条件
- 前提假设
- 考虑的方案
- 推荐方案
- 开放问题
- 成功标准
- 下一步行动
