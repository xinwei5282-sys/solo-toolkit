# Role Router

这是一个**薄路由 skill**，不维护人格内容。它的核心原则是：**先看上下文，再决定要不要用角色。**

## 何时使用

遇到以下情况时优先使用本 skill：

- 用户直接说“绘心”“明策”“龙虾”
- 用户用角色缩写，如 `ui`、`pm`、`arch`、`dev`
- 用户没有点名，但任务需要先判断该由哪类上下文主导
- 你怀疑当前回答跑偏，是因为没有先切到合适任务上下文

## 唯一角色真源

所有角色定义只认：

- `~/.openclaw/agents/<role>/agent/IDENTITY.md`
- `~/.openclaw/agents/<role>/agent/SOUL.md`
- `~/.openclaw/agents/<role>/agent/AGENTS.md`

项目内文件只保存映射关系，不复制人格内容。

角色映射表见：

- `/Users/weiran/SCRM/.claude/role-registry.md`

## 路由步骤

1. 先识别当前任务上下文：
   - 这是查询、决策、实现、评审还是验证
   - 需要哪些文件、知识、工具和状态
2. 再识别用户是否直接点名角色或隐含角色诉求
3. 到 `role-registry.md` 找到目标角色代号
4. 判断该角色在当前任务里是：
   - 不需要
   - 只需摘要视角
   - 需要完整角色真源
5. 再读取项目级 `CLAUDE.md`
6. 判断是否还要叠加某个 skill
7. 以“当前上下文 + 必要角色视角”的组合执行任务

## 主导上下文判定

- 界面、视觉、交互、样式、信息层级 -> `design`
- 需求、业务、优先级、PRD、方案取舍 -> `pm`
- 目标挑战、范围收敛、战略反问 -> `lobster`
- 开发、修复、调试、重构 -> `dev`
- 架构、边界、选型、复杂技术设计 -> `arch`

但这只是**默认映射**。如果任务已经有清晰上下文，优先遵循任务本身，而不是角色名。

## 角色接入层级

### L0：不接入角色

适用于：

- 简单问答
- 明确执行命令
- 小型代码修改
- 事实检索和精确推理

### L1：接入角色摘要

适用于：

- 需要某种判断口径
- 需要某种审美或产品视角
- 需要控制输出风格

做法：

- 只参考 `role-registry.md` 中的职责与适用任务

### L2：接入完整角色真源

适用于：

- 复杂多轮评审
- 长任务
- 明确要求持续保持某种角色工作方式

做法：

- 再读取对应 `IDENTITY.md` / `SOUL.md` / `AGENTS.md`

如果用户同时点多个角色：

- 必须先确定一个主导上下文
- 其他角色只作为补充视角
- 不要把多个角色揉成平均口吻

## 输出要求

在心里完成路由，但对用户只需简短说明：

- 当前按什么上下文主导
- 是否采用某个角色视角
- 是否补充其他角色视角

示例：

- “这次按样式优化上下文处理，采用绘心视角看层级和观感。”
- “这次按需求收敛上下文处理，先用明策视角判断取舍，不额外加载完整角色卡。”

## 边界

- 本 skill 不改角色本体
- 本 skill 不复制 `SOUL.md`
- 本 skill 不替代具体执行 skill
- 本 skill 的职责仅限：识别上下文、映射角色、按需加载、交棒
